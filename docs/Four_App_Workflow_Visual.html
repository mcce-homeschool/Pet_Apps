Kennel program · system map

Four apps, one dog record, no server

Everything is a local-first, offline PWA — no backend, no build step, all data in the browser. KennelOS is the system of record; three purpose-built companions orbit it. They stay in sync not through a database in the cloud but through the owner's own free Dropbox (a dumb file drop, one writer per file) and a single CSV hand-off. Every join is by Dog.id, so all four apps point at the exact same dog.

The hand-offs, in full

ChannelDirCarrierWhat movesThe rule that governs it
KennelOS ⇄ KennelOS ⇄ kennelos-backup.json
Dropbox		 Full backup between the owner's own phones. Merge-restore is a blind per-id upsert — don't edit the same record on both phones between a push and a pull.
KennelOS → Assistant → assistant-feed.json
Dropbox		 Allow-listed dog fields + a few event types (weight, milestone, note, preventative, medication). Privacy is enforced at feed-build time — contacts, sales, financials, parentage never reach the helper's device.
Assistant → KennelOS → assistant-outbox.json
Dropbox		 The events the helper logged, with their own UUIDs. Dry-run preview then commit; an event for an unknown dog is skipped, never invented. Re-import is a no-op.
KennelOS → Papers → KennelOS backup JSON
Dropbox		 The dog records, so Papers files documents against real dogs. One-directional, non-destructive upsert joined on Dog.id. There is no write-back into KennelOS.
Papers → KennelOS ⤍ Papers backup .zip
Dropbox, read-only		 A read-only Documents viewer inside KennelOS shows / downloads the files. Papers stays the sole owner of documents; the bytes are held in memory and never enter KennelOS's database.
Receipts → KennelOS → expense CSV Receipt & mileage rows — the extracted numbers only. receipt_number ties each ledger line back to its photo; re-importing updates rather than duplicates; the photos stay in Receipts.

What each app keeps to itself

KennelOS owns the truth

Every dog, litter, sale, contract and cost. It deliberately stores no files and no receipt photos — those live in the companions.

The helper sees a slice

KennelAssistant runs its own tiny database. Only basic dog info and loggable care events ever land on that phone — that boundary is the security model.

Papers holds the paperwork

The pedigrees, health tests and signed contracts as real PDFs. KennelOS can look but never touches — one owner for the document bytes.

Receipts keeps the photos

The receipt images and mileage log stay put; only the numbers cross into the ledger. Its own backup is the only protection for the pictures.

Transport is the owner's own Dropbox — two app-folder-scoped connections (/Apps/KennelOS/ and /Apps/Kennel Papers/), PKCE OAuth, no SDK, no backend — plus one plain CSV file for Receipts. Nothing syncs on its own; every arrow is a button the owner presses, and all four apps keep working fully offline.
